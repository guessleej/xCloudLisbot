## 1. 系統需求

| 項目 | 需求 |
|------|------|
| **瀏覽器** | Chrome 90+、Edge 90+、Safari 15+、Firefox 88+ |
| **麥克風** | 系統需授予麥克風存取權限（即時錄音功能） |

## 2. 首次登入

### 2.1 選擇登入方式

開啟系統網址後，您會看到登入頁面，支援以下四種方式：

| 登入方式 | 適用對象 | 說明 |
|---------|---------|------|
| **Microsoft** | 企業用戶、Office 365 用戶 | 使用 Microsoft Entra ID，可同時連結 Outlook 行事曆 |
| **Google** | 個人用戶、Workspace 用戶 | 支援連結 Google Calendar |
| **GitHub** | 開發者 | 使用 GitHub 帳號 |
| **Apple** | Apple 裝置用戶 | 使用 Apple ID |

### 2.2 授權流程

1. 點擊對應的登入按鈕
2. 系統彈出授權視窗，同意授予基本個人資訊存取
3. 授權完成後自動關閉視窗，頁面自動登入
4. 登入狀態儲存於瀏覽器 Session，關閉分頁後需重新登入

> ⚠️ **注意**：請確保瀏覽器未封鎖彈出視窗（popup），否則 OAuth 視窗無法開啟。

## 4. 行事曆整合

### 4.1 連結行事曆帳號

**連結 Google Calendar：**
1. 點擊頂部「📅 行事曆」按鈕
2. 在側邊欄中點擊「連結 Google 日曆」
3. 系統彈出 Google 授權視窗
4. 選擇您的 Google 帳號並點擊「允許」
5. 視窗自動關閉，側邊欄顯示綠色連線成功標誌

**連結 Outlook / Exchange：**
1. 前提：需先以 Microsoft 帳號登入系統
2. 點擊「連結 Outlook 行事曆」按鈕
3. 系統自動取得您已登入的 Microsoft 帳號的行事曆存取權限
4. 連結成功後顯示綠色標誌

### 4.2 從行事曆啟動會議

1. 連結成功後，側邊欄自動顯示今日行程清單
2. 使用日期選擇器可切換至其他日期
3. 行程卡片顯示：
   - 🕐 時間（開始～結束）
   - 👥 與會者清單
   - 🔴 **NOW**：目前進行中的會議
   - 🟡 **soon**：30 分鐘內即將開始的會議
4. 點擊行程卡片上的「**錄製**」按鈕
5. 系統自動填入會議標題和與會者資訊
6. 確認設定後點擊「開始錄音」

## 5. 即時錄音會議

### 5.1 會議設定（錄音前）

#### 選擇會議模式

| 模式 | 圖示 | 適用場景 |
|------|------|---------|
| 一般會議 | 🏢 | 日常商務會議，強調決策與待辦事項 |
| 訪談 | 🎤 | 人資訪談、用戶研究，強調問答摘要 |
| 腦力激盪 | 💡 | 創意工作坊，保留所有想法不過濾 |
| 課堂講座 | 📚 | 教學場景，產生課程重點筆記 |
| 每日站會 | ⚡ | Scrum Stand-up，輸出三件事格式 |
| 技術評審 | 🔍 | Code Review / 架構評審，強調技術決策 |
| 客戶會議 | 🤝 | 業務拜訪，強調客戶需求與後續跟進 |

#### 選擇語言

| 語言代碼 | 顯示名稱 | 說明 |
|---------|---------|------|
| `zh-TW` | 🇹🇼 繁體中文 | 台灣繁體中文（預設） |
| `nan-TW` | 🇹🇼 台語 | 台語輸入，自動以繁中輸出 |
| `hak-TW` | 🇹🇼 客語 | 客語輸入，自動以繁中輸出 |
| `en-US` | 🇺🇸 English | 美式英文 |
| `ja-JP` | 🇯🇵 日本語 | 日文 |
| `zh-CN` | 🇨🇳 简体中文 | 中國大陸簡體中文 |
| `auto` | 🌐 自動偵測 | 自動偵測語言 |

> ⚠️ 台語/客語為實驗性功能，辨識準確率低於繁中，建議搭配術語辭典使用。

#### 選擇摘要範本

點擊範本網格中的任一範本（內建 7 種 + 您自訂的範本）。每個範本決定 AI 摘要的輸出格式與重點。

#### 啟用術語辭典

勾選您建立的術語辭典，系統會將術語注入 Azure Speech 引擎以提升辨識準確率。可同時啟用多個辭典。

#### 設定說話者人數上限

拖曳滑桿設定最大說話者人數（2～10 人），有助於說話者分離的準確性。

### 5.2 開始錄音

1. 輸入會議標題（必填）
2. 完成上述設定
3. 點擊「**▶ 開始錄音**」
4. 瀏覽器彈出麥克風授權請求 → 點擊「**允許**」
5. 錄音開始，計時器啟動
6. 畫面右側即時顯示逐字稿，每位說話者以不同顏色標示

### 5.3 即時逐字稿

錄音過程中，逐字稿區域即時顯示：
- **說話者標識**：Speaker 1、Speaker 2…（顏色區分）
- **識別文字**：即時顯示轉錄內容
- **時間戳**：每個片段的開始時間

### 5.4 停止錄音

1. 點擊「**⬛ 停止錄音**」
2. 系統自動觸發 AI 摘要生成（顯示「生成中...」進度）
3. 約 10～30 秒後，摘要出現在右側面板

## 6. 音檔上傳轉錄

### 6.1 切換至上傳模式

點擊頂部「📤 上傳音檔」按鈕，介面切換至上傳模式。

### 6.2 上傳音檔

**支援格式：**

| 格式 | 副檔名 | 說明 |
|------|--------|------|
| MP3 | `.mp3` | 最常用音訊格式 |
| WAV | `.wav` | 無損音訊，辨識品質最佳 |
| MP4 | `.mp4` | 影片格式（取音訊） |
| M4A | `.m4a` | Apple 音訊格式 |
| OGG | `.ogg` | 開放格式 |
| FLAC | `.flac` | 無損壓縮 |

**大小限制**：最大 200MB

**上傳方式：**
- 拖曳音檔至拖曳區域（顯示「拖曳或點擊上傳音檔」）
- 或直接點擊拖曳區域選擇檔案

### 6.3 設定轉錄選項

上傳前需設定：
- **會議標題**（可自動由檔名填入）
- **語言**（同即時錄音選項）
- **摘要範本**

### 6.4 上傳與等待

1. 點擊「**上傳並轉錄**」
2. 進度條顯示上傳進度（0%→65%）
3. 上傳完成後系統自動提交 Azure Speech 批次轉錄
4. 進度條進入「AI 語音轉錄中...」狀態（最長等待 5 分鐘）
5. 轉錄完成後自動觸發摘要生成
6. 完整結果呈現在右側

> 💡 **提示**：較長的音檔（如 1 小時）可能需要 3～5 分鐘，請耐心等候，不要關閉頁面。

## 7. 術語辭典

術語辭典可提升特定專業詞彙的語音辨識準確率，例如公司名稱、產品名稱、技術術語等。

### 7.1 開啟術語辭典

點擊頂部「📚 術語辭典」按鈕，開啟管理 Modal。

### 7.2 建立新辭典

1. 點擊「**+ 新增辭典**」
2. 填入辭典名稱（如「產品術語」、「醫療用語」）
3. 填入描述（選填）
4. 在術語表格中逐一新增術語：
   - **原始詞語**：可能被辨識錯的詞（如「擷取」）
   - **偏好詞語**：期望辨識結果（如「Excel」）
   - **類別**：技術/業務/人名/品牌（方便分類）
5. 點擊「儲存」

### 7.3 CSV 批次匯入

若有大量術語，可使用 CSV 匯入：
1. 準備 CSV 檔（格式：`原始詞語,偏好詞語,類別`）
2. 點擊「📥 匯入 CSV」
3. 選擇 CSV 檔案
4. 術語自動匯入至當前辭典

**CSV 範例：**
```csv
擷取,Excel,工具
顧問,顧問,職稱
微軟,Microsoft,品牌
Azure,Azure,品牌
```

### 7.4 啟用/停用辭典

- 辭典清單中的開關可快速啟用/停用整個辭典
- 啟用的辭典才會在錄音時注入術語
- 可同時啟用多個辭典

### 7.5 在錄音中套用

在會議設定的「術語辭典」區域，勾選要使用的辭典後開始錄音，系統自動將術語注入 Azure Speech 引擎。

## 8. 摘要範本

### 8.1 內建範本

系統提供 7 種內建範本（唯讀，不可修改）：

| 範本 | 圖示 | 摘要重點 |
|------|------|---------|
| 一般會議紀錄 | 📋 | 目的、決策、待辦、下次議題 |
| 訪談記錄 | 🎤 | 受訪者觀點、關鍵問答 |
| 腦力激盪 | 💡 | 所有創意、反覆主題、值得深入的方向 |
| 課堂講義 | 📚 | 教學重點、概念、案例 |
| 每日站會 | ⚡ | 昨完成、今計劃、阻礙 |
| 技術評審 | 🔍 | 技術決策、風險、技術債 |
| 客戶拜訪 | 🤝 | 客戶需求、共識、跟進事項 |

### 8.2 建立自訂範本

1. 點擊頂部「📋 摘要範本」
2. 點擊「**+ 新增範本**」
3. 填寫：
   - **範本名稱**
   - **描述**（選填）
   - **圖示**（從表情符號選擇）
   - **GPT System Prompt**：自訂 AI 指令（可覆寫預設行為）
4. 點擊「儲存」

### 8.3 自訂 Prompt 範例

```
你是一位法律顧問助理。分析會議逐字稿並產生：
1. 法律風險摘要（條列式）
2. 重要法律條款提及
3. 需要律師確認的事項
4. 合約相關決議
格式使用繁體中文，專業法律語調，格式使用 Markdown。
```

## 9. 摘要結果與匯出

### 9.1 摘要面板說明

停止錄音或音檔轉錄完成後，右側面板顯示完整摘要報告：

**標頭資訊：**
- 📋 範本名稱（例：「一般會議紀錄」）
- 🌐 語言標識（例：「繁中」）
- 分享按鈕、匯出按鈕

**分頁內容：**

| 分頁 | 內容 |
|------|------|
| 📄 **摘要** | AI 生成的 Markdown 格式完整摘要 |
| ✅ **待辦事項** | 結構化待辦清單，含負責人、優先級、截止日期、類別 |
| 🔑 **決議** | 會議確認的重要決策清單 |
| 📅 **下次議題** | 建議下次會議討論的事項 |

**待辦事項卡片欄位：**
- 任務描述
- 🔴🟡🟢 優先級（高/中/低）
- 👤 負責人
- 📅 截止日期
- 🏷️ 類別（技術/業務/行政/其他）

### 9.2 匯出

| 格式 | 說明 |
|------|------|
| **↓ MD** | 下載 Markdown 格式文字檔（含完整摘要） |
| **↓ JSON** | 下載結構化 JSON 格式（含所有待辦事項、決議） |

## 10. 團隊協作分享

### 10.1 分享會議

1. 在摘要面板頂部點擊「**分享**」按鈕
2. 在「邀請成員」欄輸入對方 Email
3. 選擇權限：
   - **檢視**：對方可閱讀摘要，不可修改
   - **編輯**：對方可修改摘要內容（功能開發中）
4. 輸入邀請訊息（選填）
5. 點擊「**傳送邀請**」
6. 系統記錄分享關係（後續可至分享清單管理）

### 10.2 管理分享成員

分享 Modal 下方顯示目前已分享的成員清單：
- 成員 Email
- 分享時間
- 權限類型
- 🗑️ 撤銷按鈕

### 10.3 複製連結

點擊「📋 複製連結」按鈕，將會議摘要的直接連結複製至剪貼簿，可發送給有帳號的成員。

## 11. 常見問題

### Q1：麥克風沒有聲音/無法錄音

**解決步驟：**
1. 確認瀏覽器已授予麥克風權限（瀏覽器網址列左側鎖頭圖示 → 麥克風 → 允許）
2. 確認系統麥克風設定正常（控制台 → 聲音 → 輸入裝置）
3. 使用 Chrome 或 Edge 瀏覽器（Firefox 部分功能可能受限）
4. 重新整理頁面後再試

### Q2：逐字稿辨識準確率低

**解決方案：**
1. 建立並啟用**術語辭典**，加入常用專業詞彙
2. 選擇正確的**語言**設定（例如：會議用台語則選「台語」）
3. 使用高品質麥克風，減少背景噪音
4. 說話速度放慢，發音清晰

### Q3：行事曆無法連結

**Google Calendar 問題：**
- 確認瀏覽器未封鎖彈出視窗（允許 popup）
- 確認使用的是 Google 帳號登入

**Outlook Calendar 問題：**
- 必須先以 **Microsoft 帳號登入** XMeet AI
- 確認 Microsoft 帳號有存取 Exchange/Outlook 日曆的權限

### Q4：音檔上傳失敗

- 確認檔案格式為支援的格式（MP3/WAV/MP4/M4A/OGG/FLAC）
- 確認檔案大小未超過 **200MB**
- 確認網路連線穩定
- 若持續失敗，請嘗試將檔案轉換為 WAV 格式後重試

### Q5：摘要生成失敗或內容品質差

- 確認逐字稿文字內容足夠（至少 10 個字以上）
- 若逐字稿有大量錯誤字，建議先修正逐字稿再手動觸發摘要
- 選擇符合會議類型的**摘要範本**可提升品質
- 若是技術性會議，搭配**術語辭典**可顯著提升品質

### Q6：如何處理多語言混雜的會議？

- 選擇語言「🌐 自動偵測」
- AI 摘要預設以繁體中文輸出
- 可在自訂範本的 Prompt 中指定輸出語言

### Q7：Session 過期如何處理？

系統 JWT Token 有效期為 24 小時，過期後會自動要求重新登入。重新登入後，所有歷史會議記錄仍完整保留。
